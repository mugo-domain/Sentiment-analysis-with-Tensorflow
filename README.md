#Mobile App Sentiment Analysis using Bidirectional LSTMA 
Deep Learning NLP pipeline designed to automatically classify mobile application reviews into Positive or Negative categories. This project leverages natural language processing to understand contextual language shifts in user feedback, achieving high classification performance on imbalanced real-world data.🚀 Key FeaturesSequential Text Modeling: Utilizes a Recurrent Neural Network (RNN) architecture with a Bidirectional Long Short-Term Memory (LSTM) layer.Robust Preprocessing: Implements text tokenization, out-of-vocabulary (<OOV>) handling, and uniform padding sequencing.Overfitting Protection: Integrates $L_1$ and $L_2$ regularization alongside Dropout layers and an automated EarlyStopping callback.🛠️ Tech Stack & LibrariesLanguage: PythonDeep Learning Framework: TensorFlow / KerasData Manipulation & ML Evaluation: Pandas, NumPy, Scikit-Learn📈 Model Performance & EvaluationThe final model was evaluated on a held-out test set (20% of the dataset) consisting of 5,184 reviews.

Final MetricsTest Accuracy: 87.52% Macro Average F1-Score: 0.83

🏁 Wins & Challenges🏆 Project WinsBreaking the Baseline Bottleneck: The dataset was heavily imbalanced, with roughly 74% of the reviews being positive. A standard dense feed-forward network capped out at 75% accuracy—barely beating a naive baseline model. Upgrading to a Bidirectional LSTM smashed through this bottleneck, achieving a final accuracy of 87.52%.Contextual Sarcasm and Negation Handling: Early iterations of the model failed when a review contained terms like "fixed all the bugs", misclassifying them as negative due to the word "bugs". The sequential memory of the Bidirectional LSTM successfully learned word pairings, flipping these tricky, context-dependent reviews to positive with high confidence (71.35%).Balanced Symmetrical Errors: The final confusion matrix displayed highly symmetrical errors (322 False Positives vs. 325 False Negatives). This proved that the default classification threshold of 0.5 sat at the absolute sweet spot for stable predictions without requiring artificial class-weight injection.Perfect Early Stopping Rollback: By utilizing the Keras EarlyStopping callback with restore_best_weights=True, the model stopped training at Epoch 10 as validation loss began to plateau, automatically rolling back the weights to the absolute best iteration at Epoch 5.⚠️ Challenges OvercomeVocabulary Noise Control: Managing text tokenization across thousands of user reviews risked bloating the vocabulary with unique typos, slang, and structural noise. This was resolved by enforcing a strict num_words=5000 limit to isolate the most high-impact, repeating language patterns.Continuous vs. Binary Evaluation Conflicts: Encountered evaluation pipeline crashes due to a target-type mismatch between continuous raw probability predictions (sigmoidal floats) and hard binary classes needed by scikit-learn metrics. Resolved by implementing boolean threshold filtering (.astype(int)) to bridge Keras outputs with scikit-learn evaluation metrics.Overfitting Pressures: Due to the expressive capacity of recurrent layers, early iterations showed training accuracy continuing to climb past 91% while validation loss drifted. This was aggressively neutralized by compounding $L_1$/$L_2$ kernel weight constraints with structural Dropout layers.
